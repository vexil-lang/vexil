# Vexil Reference Implementation — Milestone D Design Spec

**Date:** 2026-03-25
**Status:** Draft
**Milestone:** D — Rust Codegen Backend

## Summary

Add a Rust code generation backend to the Vexil compiler. Two new crates:
`vexil-runtime` (bit-level I/O, Pack/Unpack traits, wire encoding primitives)
and `vexil-codegen` (takes `CompiledSchema`, emits a single `.rs` file).
The `vexilc` CLI gains a `codegen` subcommand. Generated code implements
the full wire format from the Vexil spec (§3–§4, §11–§12) except
`SCHEMA_HASH` (deferred to Milestone E).

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Codegen location | Separate `vexil-codegen` crate | Compiler stays lean; different dependency profiles; natural home for future backends |
| Serialization API | Trait-based (`Pack`/`Unpack`) with runtime crate | Bit-level packing logic is non-trivial; traits are idiomatic Rust; runtime is reusable |
| Output format | Single `.rs` file per schema | Simple, testable (golden files), sufficient pre-Milestone F |
| Generated header | "Code generated by vexilc. DO NOT EDIT." | Standard practice; includes source schema name for traceability |
| Semantic type mapping | Plain arrays/primitives | `[u8; 16]` for uuid, `[u8; 3]` for rgb, `i64` for timestamp, `[u8; 32]` for hash — zero deps, consumers can wrap |
| Map type | `BTreeMap<K,V>` | Spec requires sorted wire encoding; BTreeMap gives sorted iteration for free |
| Sub-byte container | `u8`/`i8` | Smallest standard integer; pack-time validation enforces N-bit range |
| Flags type | `pub struct F(pub u64)` + generated ops | Dependency-free bitflags; bitwise ops + constants generated |
| Recursive boxing | Codegen detects cycles, inserts `Box` | `Option<T>` and `Result<T,E>` are not heap-indirected; `Vec<T>` and `BTreeMap` are |
| Delta encoding | Stateful `Encoder`/`Decoder` structs | Per-field running previous value; only generated for types with `@delta` fields |
| Float representation | IEEE 754 (f32/f64) | Per spec §3.1; NaN canonicalization on encode; negative zero preserved |
| Testing | Golden files + runtime unit tests + hand-written round-trips | Proves codegen output shape, runtime correctness, and wire format compliance |
| SCHEMA_HASH | Deferred to Milestone E | Requires canonical form first |

## Architecture

### Crate Layout

```
crates/
  vexil-runtime/        — Bit-level reader/writer, LEB128, ZigZag, Pack/Unpack traits, errors
  vexil-codegen/        — CompiledSchema → Rust source string
  vexilc/               — CLI: parse, check, codegen subcommands
  vexil-lang/           — Unchanged (compiler library)
```

### Data Flow

```
.vexil source
    → vexil_lang::compile() → CompiledSchema
    → vexil_codegen::generate(&CompiledSchema) → String (Rust source)
    → write to output.rs
```

### Dependency Graph

```
vexil-codegen  →  vexil-lang (for IR types)
vexilc         →  vexil-lang + vexil-codegen
generated.rs   →  vexil-runtime (use at compile time)
```

`vexil-codegen` does NOT depend on `vexil-runtime`. It generates text that
references runtime types but doesn't link against them.

## `vexil-runtime` Crate

### BitWriter

Accumulates bits LSB-first, flushes to a byte buffer.

```rust
pub struct BitWriter {
    buf: Vec<u8>,
    current_byte: u8,
    bit_offset: u8,  // 0–7, position within current_byte
}
```

**Methods:**
- `write_bits(value: u64, count: u8)` — write N bits LSB-first
- `write_bool(v: bool)` — 1 bit
- `write_u8/u16/u32/u64(v)` — little-endian, full width
- `write_i8/i16/i32/i64(v)` — two's complement LE
- `write_f32(v: f32)` — IEEE 754 LE; canonicalize NaN to `0x7FC00000`
- `write_f64(v: f64)` — IEEE 754 LE; canonicalize NaN to `0x7FF8000000000000`
- `write_leb128(v: u64)` — unsigned LEB128
- `write_zigzag(v: i64, bits: u8)` — ZigZag mapping `(n << 1) ^ (n >> (bits-1))` then LEB128
- `write_bytes(data: &[u8])` — flush to byte boundary, LEB128 length prefix + raw bytes
- `write_string(s: &str)` — flush to byte boundary, LEB128 length prefix + UTF-8 bytes
- `flush_to_byte_boundary()` — zero-pad remaining bits in current byte
- `finish(self) -> Vec<u8>` — flush + return buffer

**Byte-boundary flushing rule:** `write_bytes`, `write_string`, and multi-byte
integer writes (`write_u16` and above) flush to byte boundary before writing.
Sub-byte writes (`write_bits`, `write_bool`) do not flush. This matches the
spec's rule for `optional<T>` where T is a byte-aligned type.

### BitReader

Reads bits LSB-first from a byte slice.

```rust
pub struct BitReader<'a> {
    data: &'a [u8],
    byte_pos: usize,
    bit_offset: u8,
}
```

**Methods:** Mirror of BitWriter — `read_bits(count) -> u64`, `read_bool`,
`read_u8/u16/u32/u64`, `read_i8/i16/i32/i64`, `read_f32/f64`,
`read_leb128() -> u64`, `read_zigzag(bits) -> i64`, `read_bytes() -> Vec<u8>`,
`read_string() -> String`, `flush_to_byte_boundary()`.

All read methods return `Result<T, DecodeError>`. EOF during any read returns
`DecodeError::UnexpectedEof`.

**LEB128 byte count limits:** `read_leb128` takes an optional `max_bytes: u8`
parameter. Per spec: u8→2, u16→3, u32→5, u64→10 bytes. Exceeding the limit
returns `DecodeError::InvalidVarint`.

### Traits

```rust
pub trait Pack {
    fn pack(&self, writer: &mut BitWriter) -> Result<(), EncodeError>;
}

pub trait Unpack: Sized {
    fn unpack(reader: &mut BitReader<'_>) -> Result<Self, DecodeError>;
}
```

### Error Types

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum EncodeError {
    /// Sub-byte value doesn't fit in declared bit width.
    ValueOutOfRange { field: &'static str, bits: u8 },
    /// String/bytes/array/map exceeds @limit.
    LimitExceeded { field: &'static str, limit: u64, actual: u64 },
}

#[derive(Debug, Clone, PartialEq)]
pub enum DecodeError {
    UnexpectedEof,
    InvalidUtf8,
    InvalidVarint,
    LimitExceeded { field: &'static str, limit: u64, actual: u64 },
    UnknownEnumVariant { type_name: &'static str, value: u64 },
    UnknownUnionVariant { type_name: &'static str, discriminant: u64 },
    RemovedField { ordinal: u16, removed_in: &'static str, reason: &'static str },
    InvalidValue { field: &'static str, message: String },
}
```

Both error types derive `thiserror::Error` for Display/Error impls.

### Global Limits (enforced in runtime)

Per spec §13.1:
- String/bytes: max 67,108,864 bytes (2^26)
- Array/map: max 16,777,216 elements (2^24)
- Enum ordinal: max 65,535
- Union variant ordinal: max 65,535
- Flags bit position: max 63

## `vexil-codegen` Crate

### Public API

```rust
pub fn generate(compiled: &CompiledSchema) -> Result<String, CodegenError>;

pub enum CodegenError {
    /// A type referenced by ID has no definition (unresolved import stub).
    UnresolvedType { type_id: TypeId, referenced_by: String },
}
```

### Internal Structure

```
crates/vexil-codegen/src/
  lib.rs        — pub fn generate(), orchestrates the pipeline
  types.rs      — ResolvedType → Rust type string mapping
  emit.rs       — Code emission helpers (indentation, section separators)
  message.rs    — Message struct + Pack/Unpack generation
  enum_gen.rs   — Enum type + Pack/Unpack generation
  flags.rs      — Flags struct + bitwise ops + Pack/Unpack generation
  union_gen.rs  — Union enum + Pack/Unpack generation
  newtype.rs    — Newtype wrapper + Pack/Unpack generation
  config.rs     — Config struct + Default generation
  boxing.rs     — Cycle detection for Box insertion
  delta.rs      — Stateful Encoder/Decoder struct generation
```

### `generate()` Flow

1. **Analyze type graph** — walk all types, detect cycles → build
   `needs_box: HashSet<(TypeId, usize)>` mapping (type id, field index)
   pairs that need `Box` wrapping.
2. **Emit header** — generated comment with source schema name, `use vexil_runtime::*;`
3. **Emit `SCHEMA_VERSION`** — from `compiled.annotations.version`
4. **For each declaration** in source order (`compiled.declarations`):
   - Emit section comment separator (`// ── TypeName ──`)
   - Emit tombstone comments + `TYPE_REMOVED_ORDINALS` constant (if tombstones exist)
   - Emit `@doc` as `///` doc comments
   - Emit `@deprecated` as `#[deprecated(since = "...", note = "...")]`
   - Emit type definition (struct/enum)
   - Emit `impl Pack` + `impl Unpack` (all types except config)
   - Emit `impl Default` (config only)
   - Emit `TypeEncoder`/`TypeDecoder` structs (if type has `@delta` fields)

### Type Mapping

| Vexil Type | Rust Type | Notes |
|---|---|---|
| `bool` | `bool` | 1-bit on wire |
| `u8`–`u64` | `u8`–`u64` | LE wire encoding |
| `i8`–`i64` | `i8`–`i64` | Two's complement LE |
| `f32`, `f64` | `f32`, `f64` | IEEE 754 LE, NaN canonicalized |
| `void` | `()` | 0 bits |
| `string` | `String` | LEB128 length + UTF-8 |
| `bytes` | `Vec<u8>` | LEB128 length + raw |
| `uuid` | `[u8; 16]` | 128 bits, big-endian byte order |
| `rgb` | `[u8; 3]` | 24 bits (R, G, B) |
| `timestamp` | `i64` | i64 microseconds since Unix epoch |
| `hash` | `[u8; 32]` | 256 bits, opaque |
| `uN` (sub-byte) | `u8` | Pack validates N-bit range |
| `iN` (sub-byte) | `i8` | Pack validates N-bit range |
| `optional<T>` | `Option<T>` | 1-bit presence flag |
| `array<T>` | `Vec<T>` | LEB128 count + elements |
| `map<K,V>` | `BTreeMap<K,V>` | Sorted iteration matches wire order |
| `result<T,E>` | `Result<T,E>` | 1-bit discriminant |
| `newtype Foo : T` | `pub struct Foo(pub T)` | Delegates to inner type |
| `flags F` | `pub struct F(pub u64)` | Bit constants + bitwise ops |
| `message M` | `pub struct M { pub fields... }` | Fields in ordinal order |
| `enum E` | `pub enum E { Variants... }` | Discriminant values = ordinal |
| `union U` | `pub enum U { Variants... }` | Tagged, LEB128 discriminant + length prefix |
| `config C` | `pub struct C { pub fields... }` | Default impl, no Pack/Unpack |
| Recursive `optional<T>` | `Option<Box<T>>` | Heap indirection for recursive cycles |
| Recursive `result<T,E>` | `Result<Box<T>, E>` (or `Result<T, Box<E>>`) | Heap indirection |

### Generated Code Patterns

#### Message

```rust
#[derive(Debug, Clone, PartialEq)]
pub struct Hello {
    pub name: String,
    pub age: u8,
}

impl Pack for Hello {
    fn pack(&self, w: &mut BitWriter) -> Result<(), EncodeError> {
        w.write_string(&self.name)?;
        w.write_u8(self.age)?;
        w.flush_to_byte_boundary();
        Ok(())
    }
}

impl Unpack for Hello {
    fn unpack(r: &mut BitReader<'_>) -> Result<Self, DecodeError> {
        let name = r.read_string()?;
        let age = r.read_u8()?;
        r.flush_to_byte_boundary();
        Ok(Self { name, age })
    }
}
```

**Empty message:** Pack writes `0x00` (zero content bits padded to byte boundary).
Unpack reads and discards one byte.

**Sub-byte fields:** Use `w.write_bits(value as u64, N)` / `r.read_bits(N)`.
Fields cross byte boundaries without padding. Message ends with
`flush_to_byte_boundary()`.

#### Enum

**Exhaustive, no explicit backing:**

Wire bit width = `max(ceil(log2(max_ordinal + 1)), 1)`.

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum Status {
    Active = 0,
    Inactive = 1,
}

impl Pack for Status {
    fn pack(&self, w: &mut BitWriter) -> Result<(), EncodeError> {
        w.write_bits(*self as u64, 1);  // ceil(log2(2)) = 1 bit
        Ok(())
    }
}

impl Unpack for Status {
    fn unpack(r: &mut BitReader<'_>) -> Result<Self, DecodeError> {
        let v = r.read_bits(1)?;
        match v {
            0 => Ok(Self::Active),
            1 => Ok(Self::Inactive),
            _ => Err(DecodeError::UnknownEnumVariant { type_name: "Status", value: v }),
        }
    }
}
```

**`@non_exhaustive` enum:**

Minimum 8-bit wire representation. Adds `Unknown(u64)` variant.

```rust
#[derive(Debug, Clone, PartialEq)]
#[non_exhaustive]
pub enum ClientKind {
    Renderer = 0,
    Agent = 1,
    Unknown(u64),
}
```

**Explicit backing type:**

Uses exactly the backing type's bit width (e.g., `u16` = 16 bits).

#### Flags

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct Permissions(pub u64);

impl Permissions {
    pub const READ: Self = Self(1 << 0);
    pub const WRITE: Self = Self(1 << 1);
    pub const EXECUTE: Self = Self(1 << 2);
    pub const DELETE: Self = Self(1 << 3);

    pub fn contains(self, other: Self) -> bool { self.0 & other.0 == other.0 }
    pub fn is_empty(self) -> bool { self.0 == 0 }
}

impl std::ops::BitOr for Permissions {
    type Output = Self;
    fn bitor(self, rhs: Self) -> Self { Self(self.0 | rhs.0) }
}
impl std::ops::BitAnd for Permissions {
    type Output = Self;
    fn bitand(self, rhs: Self) -> Self { Self(self.0 & rhs.0) }
}
impl std::ops::Not for Permissions {
    type Output = Self;
    fn not(self) -> Self { Self(!self.0) }
}
```

**Wire width:** Power-of-2 byte width from max bit position:
- Bits 0–7 → u8 (8 bits, participates in sub-byte packing)
- Bits 8–15 → u16 (16 bits, byte-aligned)
- Bits 16–31 → u32 (32 bits, byte-aligned)
- Bits 32–63 → u64 (64 bits, byte-aligned)

Multi-byte flags flush to byte boundary before writing.

#### Union

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum Color {
    Ansi { code: u8 },
    Rgb { r: u8, g: u8, b: u8 },
    Reset,
}

impl Pack for Color {
    fn pack(&self, w: &mut BitWriter) -> Result<(), EncodeError> {
        w.flush_to_byte_boundary();
        match self {
            Self::Ansi { code } => {
                w.write_leb128(0);                      // discriminant
                let mut payload = BitWriter::new();
                payload.write_u8(*code)?;
                payload.flush_to_byte_boundary();
                let bytes = payload.finish();
                w.write_leb128(bytes.len() as u64);     // byte length
                w.write_raw_bytes(&bytes);
            }
            Self::Rgb { r, g, b } => {
                w.write_leb128(1);
                let mut payload = BitWriter::new();
                payload.write_u8(*r)?;
                payload.write_u8(*g)?;
                payload.write_u8(*b)?;
                payload.flush_to_byte_boundary();
                let bytes = payload.finish();
                w.write_leb128(bytes.len() as u64);
                w.write_raw_bytes(&bytes);
            }
            Self::Reset => {
                w.write_leb128(2);
                w.write_leb128(0);                      // zero-length payload
            }
        }
        Ok(())
    }
}
```

**`@non_exhaustive` union:** Adds `Unknown { discriminant: u64, data: Vec<u8> }` variant.
Decoder reads discriminant + byte length, skips unknown payload.

#### Newtype

```rust
#[derive(Debug, Clone, PartialEq)]
pub struct UserId(pub u64);

impl Pack for UserId {
    fn pack(&self, w: &mut BitWriter) -> Result<(), EncodeError> {
        w.write_u64(self.0)?;
        Ok(())
    }
}

impl Unpack for UserId {
    fn unpack(r: &mut BitReader<'_>) -> Result<Self, DecodeError> {
        Ok(Self(r.read_u64()?))
    }
}
```

#### Config

```rust
#[derive(Debug, Clone, PartialEq)]
pub struct ServerConfig {
    pub max_sessions: u32,
    pub timeout_ms: u64,
    pub enable_tls: bool,
    pub log_level: String,
}

impl Default for ServerConfig {
    fn default() -> Self {
        Self {
            max_sessions: 1024,
            timeout_ms: 30000,
            enable_tls: false,
            log_level: String::from("info"),
        }
    }
}
```

No `Pack`/`Unpack` generated for config types.

#### Recursive Types

```rust
#[derive(Debug, Clone, PartialEq)]
pub struct Node {
    pub value: i32,
    pub next: Option<Box<Node>>,  // Box inserted for recursive cycle
}
```

Boxing detection: walk type graph, track stack of TypeIds. When reaching
`Named(id)` through `Option<T>` or `Result<T,E>` where `id` is on the stack,
mark that field for boxing. `Vec<T>` and `BTreeMap<K,V>` are already heap-
allocated, so no Box needed.

#### Delta Encoding

For messages with `@delta` fields, generate stateful encoder/decoder:

```rust
pub struct HelloEncoder {
    prev_timestamp: i64,
}

impl HelloEncoder {
    pub fn new() -> Self {
        Self { prev_timestamp: 0 }
    }

    pub fn pack(&mut self, val: &Hello, w: &mut BitWriter) -> Result<(), EncodeError> {
        // non-delta fields use stateless encoding
        w.write_string(&val.name)?;
        // delta field: compute diff, encode, update state
        let delta = val.timestamp.wrapping_sub(self.prev_timestamp);
        w.write_i64(delta)?;  // or zigzag/varint if annotated
        self.prev_timestamp = val.timestamp;
        w.flush_to_byte_boundary();
        Ok(())
    }

    pub fn reset(&mut self) {
        self.prev_timestamp = 0;
    }
}

pub struct HelloDecoder {
    prev_timestamp: i64,
}

impl HelloDecoder {
    pub fn new() -> Self {
        Self { prev_timestamp: 0 }
    }

    pub fn unpack(&mut self, r: &mut BitReader<'_>) -> Result<Hello, DecodeError> {
        let name = r.read_string()?;
        let delta = r.read_i64()?;
        let timestamp = self.prev_timestamp.wrapping_add(delta);
        self.prev_timestamp = timestamp;
        r.flush_to_byte_boundary();
        Ok(Hello { name, timestamp })
    }

    pub fn reset(&mut self) {
        self.prev_timestamp = 0;
    }
}
```

Messages without `@delta` fields use stateless `Pack`/`Unpack` only.

#### Annotations in Generated Code

**`@doc`:** Emitted as `///` doc comments before the type or field.

**`@deprecated`:** Emitted as `#[deprecated(since = "...", note = "...")]`
on the field. If `since` is absent, only `note` is emitted.

**`@since`:** Emitted as a doc comment: `/// @since 1.0.0`.

**`@non_exhaustive`:** Emitted as `#[non_exhaustive]` on enums and union enums.

**Tombstones:** Emitted as comments and a constant per type:
```rust
// REMOVED @2 (since 0.9.0): Replaced by name
pub const HELLO_REMOVED_ORDINALS: &[(u16, &str, &str)] = &[
    (2, "0.9.0", "Replaced by name"),
];
```

## IR Additions (Prerequisites)

Two small additions to the IR to support correct codegen:

### Enum Wire Bits

Add `wire_bits: u8` field to `EnumDef`. Computed during lowering/typeck:
- If explicit backing type: `U8→8, U16→16, U32→32, U64→64`
- If `@non_exhaustive` and no explicit backing: `max(ceil(log2(max_ordinal + 1)), 8)`
- Otherwise: `max(ceil(log2(max_ordinal + 1)), 1)`

### Flags Wire Width

Add `wire_bytes: u8` field to `FlagsDef`. Computed from max bit position:
- Max bit 0–7 → 1 (u8)
- Max bit 8–15 → 2 (u16)
- Max bit 16–31 → 4 (u32)
- Max bit 32–63 → 8 (u64)
- No bits defined → 1 (u8, minimum)

Both are computed in `typeck.rs` alongside wire size computation.

## Testing Strategy

### Layer 1: `vexil-runtime` Unit Tests

In-crate tests for bit-level I/O primitives:

- BitWriter/BitReader round-trips: single bits, sub-byte, cross-byte-boundary,
  byte-boundary flushing
- LSB-first correctness: spec's worked example (u3 + u5 + u6 → exact bytes)
- LEB128: encode/decode, byte count limits, edge cases (0, max values)
- ZigZag: mapping correctness (0→0, -1→1, 1→2, -2→3), round-trip
- String/bytes: length prefix, UTF-8 validation, global limit (2^26)
- Float: NaN canonicalization, negative zero preservation
- Empty message: single 0x00 byte

### Layer 2: `vexil-codegen` Golden File Tests

Compile corpus `.vexil` → generate Rust → compare against checked-in snapshots:

```
crates/vexil-codegen/tests/golden/
  006_message.rs
  007_enum.rs
  008_flags.rs
  009_union.rs
  010_newtype.rs
  011_config.rs
  016_recursive.rs
```

Test runner does string comparison, reports diff on failure.

### Layer 3: Wire Format Round-Trip Tests

Hand-written Rust types + `Pack`/`Unpack` impls mirroring codegen output.
Tests prove wire format correctness without compiling generated code:

- Construct value → pack → unpack → assert equality
- Pack → assert exact byte sequence against spec-derived expected values
- Sub-byte packing: u3 + u5 + u6 produces `[0b_sssss_kkk, 0b_00_eeeeee]`
- Union wire format: discriminant + length prefix + payload
- Optional with byte-boundary flushing
- Delta encoding: two sequential values, verify delta on wire
- `@limit` enforcement on encode and decode
- `@non_exhaustive` enum: decode unknown → `Unknown(n)`
- `@non_exhaustive` union: decode unknown → skip via length prefix
- Decode invalid input → correct error variant

## Scope Boundaries

### In Scope

- `vexil-runtime` crate (BitWriter, BitReader, Pack, Unpack, errors, LEB128, ZigZag)
- `vexil-codegen` crate (CompiledSchema → Rust source string)
- `vexilc codegen` subcommand
- All wire encoding rules from spec §3–§4
- All encoding annotations: `@varint`, `@zigzag`, `@delta`, `@limit`
- All codegen contract items from §11.2 except `SCHEMA_HASH`
- Tombstone comments + `REMOVED_ORDINALS` constants
- `@deprecated`, `@non_exhaustive`, `@doc`, `@since` in generated output
- Box insertion for recursive types through optional/result
- IR additions: `EnumDef.wire_bits`, `FlagsDef.wire_bytes`
- Three test layers

### Out of Scope (Deferred)

- `SCHEMA_HASH` — Milestone E (canonical form + BLAKE3)
- Multi-file codegen (imported types) — Milestone F
- Other language backends — future
- `no_std` runtime support — future (design for it, don't implement)
- `@type`/`@domain`/`@revision` — VNP protocol-level; preserved as doc comments only
