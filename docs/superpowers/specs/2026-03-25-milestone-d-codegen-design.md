# Vexil Reference Implementation — Milestone D Design Spec

**Date:** 2026-03-25
**Status:** Draft
**Milestone:** D — Rust Codegen Backend

## Summary

Add a Rust code generation backend to the Vexil compiler. Two new crates:
`vexil-runtime` (bit-level I/O, Pack/Unpack traits, wire encoding primitives)
and `vexil-codegen` (takes `CompiledSchema`, emits a single `.rs` file).
The `vexilc` CLI gains a `codegen` subcommand. Generated code implements
the full wire format from the Vexil spec (§3–§4, §11–§12) except
`SCHEMA_HASH` (deferred to Milestone E).

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Codegen location | Separate `vexil-codegen` crate | Compiler stays lean; different dependency profiles; natural home for future backends |
| Serialization API | Trait-based (`Pack`/`Unpack`) with runtime crate | Bit-level packing logic is non-trivial; traits are idiomatic Rust; runtime is reusable |
| Output format | Single `.rs` file per schema | Simple, testable (golden files), sufficient pre-Milestone F |
| Generated header | "Code generated by vexilc. DO NOT EDIT." | Standard practice; includes source schema name for traceability |
| Semantic type mapping | Per spec §3.3 | `(u8,u8,u8)` for rgb, `[u8; 16]` for uuid, `i64` for timestamp, `[u8; 32]` for hash |
| Map type | `BTreeMap<K,V>` | Spec requires sorted wire encoding; BTreeMap gives sorted iteration for free |
| Sub-byte container | `u8`/`i8` | Smallest standard integer; pack-time validation enforces N-bit range |
| Flags type | `pub struct F(pub u64)` + generated ops | Dependency-free bitflags; bitwise ops + constants generated |
| Recursive boxing | Codegen detects cycles, inserts `Box` | `Option<T>` and `Result<T,E>` are not heap-indirected; `Vec<T>` and `BTreeMap` are |
| Delta encoding | Stateful `Encoder`/`Decoder` structs | Per-field running previous value; only generated for types with `@delta` fields |
| Float representation | IEEE 754 (f32/f64) | Per spec §3.1; NaN canonicalization on encode; negative zero preserved |
| Testing | Golden files + runtime unit tests + hand-written round-trips | Proves codegen output shape, runtime correctness, and wire format compliance |
| SCHEMA_HASH | Deferred to Milestone E | Requires canonical form first |

## Architecture

### Crate Layout

```
crates/
  vexil-runtime/        — Bit-level reader/writer, LEB128, ZigZag, Pack/Unpack traits, errors
  vexil-codegen/        — CompiledSchema → Rust source string
  vexilc/               — CLI: parse, check, codegen subcommands
  vexil-lang/           — Unchanged (compiler library)
```

### Data Flow

```
.vexil source
    → vexil_lang::compile() → CompiledSchema
    → vexil_codegen::generate(&CompiledSchema) → String (Rust source)
    → write to output.rs
```

### Dependency Graph

```
vexil-codegen  →  vexil-lang (for IR types)
vexilc         →  vexil-lang + vexil-codegen
generated.rs   →  vexil-runtime (use at compile time)
```

`vexil-codegen` does NOT depend on `vexil-runtime`. It generates text that
references runtime types but doesn't link against them.

## `vexil-runtime` Crate

### BitWriter

Accumulates bits LSB-first, flushes to a byte buffer.

```rust
pub struct BitWriter {
    buf: Vec<u8>,
    current_byte: u8,
    bit_offset: u8,  // 0–7, position within current_byte
}
```

**Methods (all infallible unless noted):**
- `write_bits(value: u64, count: u8)` — write N bits LSB-first
- `write_bool(v: bool)` — 1 bit
- `write_u8(v: u8)` — 8 bits LE (flushes to byte boundary first)
- `write_u16/u32/u64(v)` — LE, full width (flushes to byte boundary first)
- `write_i8/i16/i32/i64(v)` — two's complement LE (flushes to byte boundary first)
- `write_f32(v: f32)` — IEEE 754 LE; canonicalize NaN to `0x7FC00000` (flushes first)
- `write_f64(v: f64)` — IEEE 754 LE; canonicalize NaN to `0x7FF8000000000000` (flushes first)
- `write_leb128(v: u64)` — unsigned LEB128 (must be at byte boundary)
- `write_zigzag(v: i64, type_bits: u8)` — ZigZag mapping then LEB128. `type_bits` is the source integer width (8, 16, 32, or 64) used for the arithmetic shift in the formula `(n << 1) ^ (n >> (type_bits - 1))`.
- `write_bytes(data: &[u8])` — flush to byte boundary, LEB128 length prefix (max 4 LEB128 bytes) + raw bytes
- `write_string(s: &str)` — flush to byte boundary, LEB128 length prefix (max 4 LEB128 bytes) + UTF-8 bytes
- `write_raw_bytes(data: &[u8])` — write raw bytes without length prefix (must be at byte boundary). Used for union payload emission.
- `flush_to_byte_boundary()` — zero-pad remaining bits in current byte. Returns `()` (infallible).
- `finish(self) -> Vec<u8>` — flush + return buffer

**Byte-boundary flushing rule:** Multi-byte integer writes (`write_u8` and above,
`write_i8` and above, `write_f32/f64`), `write_bytes`, `write_string`,
`write_leb128`, and `write_raw_bytes` all flush to byte boundary before writing.
Sub-byte writes (`write_bits`, `write_bool`) do NOT flush. This matches the
spec's rule: sub-byte fields pack continuously, while byte-aligned types
require alignment.

### BitReader

Reads bits LSB-first from a byte slice. Tracks recursion depth for nested
type decoding.

```rust
pub struct BitReader<'a> {
    data: &'a [u8],
    byte_pos: usize,
    bit_offset: u8,
    recursion_depth: u32,
}
```

**Methods:** All return `Result<T, DecodeError>` except `flush_to_byte_boundary()`.

- `read_bits(count: u8) -> Result<u64, DecodeError>` — read N bits LSB-first
- `read_bool() -> Result<bool, DecodeError>` — 1 bit
- `read_u8/u16/u32/u64() -> Result<T, DecodeError>` — LE (flushes to byte boundary first)
- `read_i8/i16/i32/i64() -> Result<T, DecodeError>` — two's complement LE
- `read_f32/f64() -> Result<T, DecodeError>` — IEEE 754 LE
- `read_leb128(max_bytes: u8) -> Result<u64, DecodeError>` — unsigned LEB128 with byte count limit. MUST reject overlong encodings (trailing zero-value continuation bytes that don't alter the decoded value). Returns `DecodeError::InvalidVarint` on overlong or over-limit sequences.
- `read_zigzag(type_bits: u8, max_bytes: u8) -> Result<i64, DecodeError>` — read LEB128, reverse ZigZag mapping
- `read_bytes() -> Result<Vec<u8>, DecodeError>` — flush, read LEB128 length (max 4 LEB128 bytes), read raw bytes. Reject if decoded length > 2^26.
- `read_string() -> Result<String, DecodeError>` — like `read_bytes` but validates UTF-8
- `read_raw_bytes(len: usize) -> Result<Vec<u8>, DecodeError>` — read exactly `len` bytes (must be at byte boundary)
- `flush_to_byte_boundary()` — skip remaining bits in current byte. Returns `()` (infallible).
- `enter_recursive() -> Result<(), DecodeError>` — increment depth counter, return `DecodeError::RecursionLimitExceeded` if depth > 64
- `leave_recursive()` — decrement depth counter

**LEB128 length prefix limit:** For `string`, `bytes`, `array`, and `map`
length prefixes, the LEB128 encoding MUST NOT exceed 4 bytes (sufficient for
values up to 2^28). This is a distinct check from the value-range limit.
`read_leb128(4)` is used for all length prefixes.

### Traits

```rust
pub trait Pack {
    fn pack(&self, writer: &mut BitWriter) -> Result<(), EncodeError>;
}

pub trait Unpack: Sized {
    fn unpack(reader: &mut BitReader<'_>) -> Result<Self, DecodeError>;
}
```

### Error Types

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum EncodeError {
    /// Sub-byte value doesn't fit in declared bit width.
    ValueOutOfRange { field: &'static str, bits: u8 },
    /// String/bytes/array/map exceeds @limit.
    LimitExceeded { field: &'static str, limit: u64, actual: u64 },
}

#[derive(Debug, Clone, PartialEq)]
pub enum DecodeError {
    UnexpectedEof,
    InvalidUtf8,
    /// Overlong LEB128, exceeded max byte count, or non-canonical encoding.
    InvalidVarint,
    LimitExceeded { field: &'static str, limit: u64, actual: u64 },
    UnknownEnumVariant { type_name: &'static str, value: u64 },
    UnknownUnionVariant { type_name: &'static str, discriminant: u64 },
    RemovedField { ordinal: u16, removed_in: &'static str, reason: &'static str },
    InvalidValue { field: &'static str, message: String },
    /// Recursive type nesting exceeded 64 levels (spec §13.1).
    RecursionLimitExceeded,
    /// Schema hash mismatch (variant defined now, populated by Milestone E).
    SchemaMismatch { local: [u8; 32], remote: [u8; 32] },
}
```

Both error types derive `thiserror::Error` for Display/Error impls.

`DecodeError::SchemaMismatch` is defined in the type now but not emitted by
any generated code until Milestone E adds `SCHEMA_HASH` support.

### Global Limits (enforced in runtime)

Per spec §13.1:
- String/bytes: max 67,108,864 bytes (2^26)
- Array/map: max 16,777,216 elements (2^24)
- Enum ordinal: max 65,535
- Union variant ordinal: max 65,535
- Flags bit position: max 63
- LEB128 length prefix: max 4 bytes for string/bytes/array/map counts
- Recursive nesting depth: max 64 levels

## `vexil-codegen` Crate

### Public API

```rust
pub fn generate(compiled: &CompiledSchema) -> Result<String, CodegenError>;

pub enum CodegenError {
    /// A type referenced by ID has no definition (unresolved import stub).
    UnresolvedType { type_id: TypeId, referenced_by: String },
}
```

### Internal Structure

```
crates/vexil-codegen/src/
  lib.rs        — pub fn generate(), orchestrates the pipeline
  types.rs      — ResolvedType → Rust type string mapping
  emit.rs       — Code emission helpers (indentation, section separators)
  message.rs    — Message struct + Pack/Unpack generation
  enum_gen.rs   — Enum type + Pack/Unpack generation
  flags.rs      — Flags struct + bitwise ops + Pack/Unpack generation
  union_gen.rs  — Union enum + Pack/Unpack generation
  newtype.rs    — Newtype wrapper + Pack/Unpack generation
  config.rs     — Config struct + Default generation
  boxing.rs     — Cycle detection for Box insertion
  delta.rs      — Stateful Encoder/Decoder struct generation
```

### `generate()` Flow

1. **Analyze type graph** — walk all types, detect cycles → build
   `needs_box: HashSet<(TypeId, usize)>` mapping (type id, field index)
   pairs that need `Box` wrapping.
2. **Emit header** — generated comment with source schema name, `use vexil_runtime::*;`
3. **Emit `SCHEMA_VERSION`** — from `compiled.annotations.version`
4. **For each declaration** in source order (`compiled.declarations`):
   - Emit section comment separator (`// ── TypeName ──`)
   - Emit tombstone comments + `REMOVED_ORDINALS` constant (if tombstones exist)
   - Emit `@doc` as `///` doc comments
   - Emit `@deprecated` as `#[deprecated(...)]` (on types and fields)
   - Emit type definition (struct/enum)
   - Emit `impl Pack` + `impl Unpack` (all types except config)
   - Emit `impl Default` (config only)
   - Emit `TypeEncoder`/`TypeDecoder` structs (if type has `@delta` fields)

### Type Mapping

| Vexil Type | Rust Type | Notes |
|---|---|---|
| `bool` | `bool` | 1-bit on wire |
| `u8`–`u64` | `u8`–`u64` | LE wire encoding |
| `i8`–`i64` | `i8`–`i64` | Two's complement LE |
| `f32`, `f64` | `f32`, `f64` | IEEE 754 LE, NaN canonicalized |
| `void` | `()` | 0 bits |
| `string` | `String` | LEB128 length + UTF-8 |
| `bytes` | `Vec<u8>` | LEB128 length + raw |
| `uuid` | `[u8; 16]` | 128 bits, big-endian byte order |
| `rgb` | `(u8, u8, u8)` | 24 bits (R, G, B) — per spec §3.3 |
| `timestamp` | `i64` | i64 microseconds since Unix epoch |
| `hash` | `[u8; 32]` | 256 bits, opaque |
| `uN` (sub-byte) | `u8` | Pack validates N-bit range |
| `iN` (sub-byte) | `i8` | Pack validates N-bit range |
| `optional<T>` | `Option<T>` | 1-bit presence flag |
| `array<T>` | `Vec<T>` | LEB128 count + elements |
| `map<K,V>` | `BTreeMap<K,V>` | Sorted iteration matches wire order |
| `result<T,E>` | `Result<T,E>` | 1-bit discriminant |
| `newtype Foo : T` | `pub struct Foo(pub T)` | Delegates to inner type |
| `flags F` | `pub struct F(pub u64)` | Bit constants + bitwise ops |
| `message M` | `pub struct M { pub fields... }` | Fields in ordinal order |
| `enum E` | `pub enum E { Variants... }` | Discriminant values = ordinal |
| `union U` | `pub enum U { Variants... }` | Tagged, LEB128 discriminant + length prefix |
| `config C` | `pub struct C { pub fields... }` | Default impl, no Pack/Unpack |
| Recursive `optional<T>` | `Option<Box<T>>` | Heap indirection for recursive cycles |
| Recursive `result<T,E>` | `Result<Box<T>, E>` (or `Result<T, Box<E>>`) | Heap indirection |

### Generated Code Patterns

#### Message

```rust
#[derive(Debug, Clone, PartialEq)]
pub struct Hello {
    pub name: String,
    pub age: u8,
}

impl Pack for Hello {
    fn pack(&self, w: &mut BitWriter) -> Result<(), EncodeError> {
        w.write_string(&self.name)?;
        w.write_u8(self.age);
        w.flush_to_byte_boundary();
        Ok(())
    }
}

impl Unpack for Hello {
    fn unpack(r: &mut BitReader<'_>) -> Result<Self, DecodeError> {
        let name = r.read_string()?;
        let age = r.read_u8()?;
        r.flush_to_byte_boundary();
        Ok(Self { name, age })
    }
}
```

**Empty message:** `flush_to_byte_boundary()` on an empty bit stream produces
a single `0x00` byte (zero content bits padded to byte boundary per spec §4.1).
Both pack and unpack call `flush_to_byte_boundary()` — no special case needed.

**Sub-byte fields:** Use `w.write_bits(value as u64, N)` / `r.read_bits(N)`.
Fields cross byte boundaries without padding. Message ends with
`flush_to_byte_boundary()`.

#### Enum

**Exhaustive, no explicit backing:**

Wire bit width = `max(ceil(log2(max_ordinal + 1)), 1)`. Stored in
`EnumDef.wire_bits` (see IR Additions section).

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum Status {
    Active = 0,
    Inactive = 1,
}

impl Pack for Status {
    fn pack(&self, w: &mut BitWriter) -> Result<(), EncodeError> {
        w.write_bits(*self as u64, 1);  // wire_bits = 1
        Ok(())
    }
}

impl Unpack for Status {
    fn unpack(r: &mut BitReader<'_>) -> Result<Self, DecodeError> {
        let v = r.read_bits(1)?;
        match v {
            0 => Ok(Self::Active),
            1 => Ok(Self::Inactive),
            _ => Err(DecodeError::UnknownEnumVariant { type_name: "Status", value: v }),
        }
    }
}
```

**`@non_exhaustive` enum:**

Minimum 8-bit wire representation. Adds `Unknown(u64)` variant.

```rust
#[derive(Debug, Clone, PartialEq)]
#[non_exhaustive]
pub enum ClientKind {
    Renderer = 0,
    Agent = 1,
    Unknown(u64),
}
```

**Explicit backing type:**

Uses exactly the backing type's bit width (e.g., `u16` = 16 bits).

#### Flags

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct Permissions(pub u64);

impl Permissions {
    pub const READ: Self = Self(1 << 0);
    pub const WRITE: Self = Self(1 << 1);
    pub const EXECUTE: Self = Self(1 << 2);
    pub const DELETE: Self = Self(1 << 3);

    pub fn contains(self, other: Self) -> bool { self.0 & other.0 == other.0 }
    pub fn is_empty(self) -> bool { self.0 == 0 }
}

impl std::ops::BitOr for Permissions {
    type Output = Self;
    fn bitor(self, rhs: Self) -> Self { Self(self.0 | rhs.0) }
}
impl std::ops::BitAnd for Permissions {
    type Output = Self;
    fn bitand(self, rhs: Self) -> Self { Self(self.0 & rhs.0) }
}
impl std::ops::Not for Permissions {
    type Output = Self;
    fn not(self) -> Self { Self(!self.0) }
}
```

**Wire width:** Power-of-2 byte width from max bit position, stored in
`FlagsDef.wire_bytes` (see IR Additions):
- Bits 0–7 → 1 byte / u8 (participates in sub-byte packing)
- Bits 8–15 → 2 bytes / u16 (byte-aligned)
- Bits 16–31 → 4 bytes / u32 (byte-aligned)
- Bits 32–63 → 8 bytes / u64 (byte-aligned)

Multi-byte flags flush to byte boundary before writing.

#### Union

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum Color {
    Ansi { code: u8 },
    Rgb { r: u8, g: u8, b: u8 },
    Reset,
}

impl Pack for Color {
    fn pack(&self, w: &mut BitWriter) -> Result<(), EncodeError> {
        w.flush_to_byte_boundary();
        match self {
            Self::Ansi { code } => {
                w.write_leb128(0);                      // discriminant
                let mut payload = BitWriter::new();
                payload.write_u8(*code);
                payload.flush_to_byte_boundary();
                let bytes = payload.finish();
                w.write_leb128(bytes.len() as u64);     // byte length
                w.write_raw_bytes(&bytes);               // payload (no length prefix)
            }
            Self::Rgb { r, g, b } => {
                w.write_leb128(1);
                let mut payload = BitWriter::new();
                payload.write_u8(*r);
                payload.write_u8(*g);
                payload.write_u8(*b);
                payload.flush_to_byte_boundary();
                let bytes = payload.finish();
                w.write_leb128(bytes.len() as u64);
                w.write_raw_bytes(&bytes);
            }
            Self::Reset => {
                w.write_leb128(2);
                w.write_leb128(0);                      // zero-length payload
            }
        }
        Ok(())
    }
}
```

**`@non_exhaustive` union:** Adds `Unknown { discriminant: u64, data: Vec<u8> }` variant.
Decoder reads discriminant + byte length, skips unknown payload via `read_raw_bytes`.

#### Optional

Generated code depends on whether `T` is byte-aligned or sub-byte.

**`optional<T>` where T is byte-aligned** (message, union, string, bytes, semantic, multi-byte integer):

```rust
// optional<string> field in a message
// Pack:
w.write_bool(self.name.is_some());
if let Some(ref val) = self.name {
    w.flush_to_byte_boundary();  // flush before byte-aligned T
    w.write_string(val)?;
}
// Unpack:
let has_name = r.read_bool()?;
let name = if has_name {
    r.flush_to_byte_boundary();  // flush before byte-aligned T
    Some(r.read_string()?)
} else {
    None
};
```

**`optional<T>` where T is sub-byte** (bool, uN, iN, sub-byte enum):

```rust
// optional<u3> field
// Pack:
w.write_bool(self.val.is_some());
if let Some(val) = self.val {
    w.write_bits(val as u64, 3);  // no flush, sub-byte packing continues
}
// Unpack:
let has_val = r.read_bool()?;
let val = if has_val {
    Some(r.read_bits(3)? as u8)   // no flush
} else {
    None
};
```

#### Result

```rust
// result<u32, string> field
// Pack:
match &self.outcome {
    Ok(val) => {
        w.write_bool(false);  // 0 = Ok
        w.write_u32(*val);
    }
    Err(val) => {
        w.write_bool(true);   // 1 = Err
        w.write_string(val)?;
    }
}
// Unpack:
let is_err = r.read_bool()?;
let outcome = if !is_err {
    Ok(r.read_u32()?)
} else {
    Err(r.read_string()?)
};
```

The 1-bit discriminant participates in sub-byte packing. The same byte-boundary
flushing rules as `optional<T>` apply to the Ok/Err payloads based on whether
T/E are byte-aligned or sub-byte.

When T is `void`, Ok writes only the discriminant (0 additional bits).
When E is `void`, Err writes only the discriminant.

#### Recursive Types

```rust
#[derive(Debug, Clone, PartialEq)]
pub struct Node {
    pub value: i32,
    pub next: Option<Box<Node>>,  // Box inserted for recursive cycle
}
```

Boxing detection: walk type graph, track stack of TypeIds. When reaching
`Named(id)` through `Option<T>` or `Result<T,E>` where `id` is on the stack,
mark that field for boxing. `Vec<T>` and `BTreeMap<K,V>` are already heap-
allocated, so no Box needed.

Generated Pack/Unpack for recursive types call `r.enter_recursive()?` before
descending into the nested type and `r.leave_recursive()` after, enforcing the
64-level depth limit from spec §13.1.

#### Delta Encoding

For messages with `@delta` fields, generate stateful encoder/decoder. The
delta field type determines the operation:

**Integer delta (`@delta` on u8–u64, i8–i64, uN, iN):**
- Unsigned: `delta = current.wrapping_sub(previous)` (unsigned wrapping subtraction)
- Signed: `delta = current.wrapping_sub(previous)` (signed wrapping subtraction)
- Encode `delta` using the field's base encoding (raw, `@varint`, or `@zigzag`)

**Float delta (`@delta` on f32, f64):**
- `delta = current - previous` (standard IEEE 754 subtraction)
- Initial previous value is `+0.0` (positive zero)
- The encoded delta is subject to NaN canonicalization (same as non-delta floats)
- Delta is encoded using the field's base encoding (always `Default` for floats since `@varint`/`@zigzag` are not valid on floats)

**Composition with encoding annotations:**

`@delta @varint u32`:
```rust
// Encoder:
let delta = val.counter.wrapping_sub(self.prev_counter);
w.write_leb128(delta as u64);  // varint encodes the unsigned delta
self.prev_counter = val.counter;

// Decoder:
let delta = r.read_leb128(5)? as u32;  // max 5 bytes for u32
let counter = self.prev_counter.wrapping_add(delta);
self.prev_counter = counter;
```

`@delta @zigzag i32`:
```rust
// Encoder:
let delta = val.offset.wrapping_sub(self.prev_offset);
w.write_zigzag(delta as i64, 32);  // zigzag encodes the signed delta
self.prev_offset = val.offset;

// Decoder:
let delta = r.read_zigzag(32, 5)? as i32;  // max 5 bytes for i32
let offset = self.prev_offset.wrapping_add(delta);
self.prev_offset = offset;
```

**Full stateful encoder/decoder example:**

```rust
pub struct HelloEncoder {
    prev_timestamp: i64,
}

impl HelloEncoder {
    pub fn new() -> Self {
        Self { prev_timestamp: 0 }
    }

    pub fn pack(&mut self, val: &Hello, w: &mut BitWriter) -> Result<(), EncodeError> {
        w.write_string(&val.name)?;
        let delta = val.timestamp.wrapping_sub(self.prev_timestamp);
        w.write_i64(delta);
        self.prev_timestamp = val.timestamp;
        w.flush_to_byte_boundary();
        Ok(())
    }

    pub fn reset(&mut self) {
        self.prev_timestamp = 0;
    }
}

pub struct HelloDecoder {
    prev_timestamp: i64,
}

impl HelloDecoder {
    pub fn new() -> Self {
        Self { prev_timestamp: 0 }
    }

    pub fn unpack(&mut self, r: &mut BitReader<'_>) -> Result<Hello, DecodeError> {
        let name = r.read_string()?;
        let delta = r.read_i64()?;
        let timestamp = self.prev_timestamp.wrapping_add(delta);
        self.prev_timestamp = timestamp;
        r.flush_to_byte_boundary();
        Ok(Hello { name, timestamp })
    }

    pub fn reset(&mut self) {
        self.prev_timestamp = 0;
    }
}
```

Messages without `@delta` fields use stateless `Pack`/`Unpack` only — no
encoder/decoder structs are generated.

#### Annotations in Generated Code

**`@doc`:** Emitted as `///` doc comments before the type or field.

**`@deprecated` on declarations:** Emitted as `#[deprecated(since = "...", note = "...")]`
on the struct/enum itself. Both declaration-level and field-level `@deprecated`
produce the same Rust attribute.

**`@deprecated` on fields:** Emitted as `#[deprecated(since = "...", note = "...")]`
on the field. If `since` is absent, only `note` is emitted:
`#[deprecated(note = "reason")]`.

**`@since`:** Emitted as a `///` doc comment: `/// @since 1.0.0`. This is the
chosen form for the Rust backend (the spec requires inclusion in generated
documentation but does not mandate a specific syntax).

**`@non_exhaustive`:** Emitted as `#[non_exhaustive]` on enums and union enums.

**`@type`, `@domain`, `@revision`:** These are VNP protocol-level annotations.
They are NOT stored in `ResolvedAnnotations` (which only captures common
annotations). The codegen emits them as doc comments on the type:
`/// @type(0x01) @domain(Handshake) @revision(1)`. The raw annotation data is
available from the AST (which is returned alongside the IR in `CompileResult`)
if needed, but codegen reads from the IR only.

**Tombstones:** Emitted as comments and a per-type `REMOVED_ORDINALS` constant:
```rust
// REMOVED @2 (since 0.9.0): Replaced by name
pub const REMOVED_ORDINALS: &[(u16, &str, &str)] = &[
    (2, "0.9.0", "Replaced by name"),
];
```

Per spec §12.3, the constant is named `REMOVED_ORDINALS`. When multiple types
in the same file have tombstones, each constant is scoped inside its own
`impl TypeName` block or prefixed with the type name to avoid collision:
`pub const HELLO_REMOVED_ORDINALS`. The spec's example uses the bare name
within a single-type context; the codegen uses the prefixed form for
unambiguous multi-type files.

## IR Additions (Prerequisites)

### Enum Wire Bits

Add `wire_bits: u8` field to `EnumDef`. This field augments the existing
`backing: EnumBacking` field (which records the explicit backing type or
defaults to U32). `wire_bits` stores the actual bit count used on the wire:

- If explicit backing type specified: `U8→8, U16→16, U32→32, U64→64`
- If `@non_exhaustive` and no explicit backing: `max(ceil(log2(max_ordinal + 1)), 8)`
- Otherwise (exhaustive, no explicit backing): `max(ceil(log2(max_ordinal + 1)), 1)`

`backing` is kept for codegen to know the explicit type choice (affects
generated code comments and type assertions). `wire_bits` is the value
used for `write_bits`/`read_bits` calls.

Computed in `typeck.rs` alongside wire size computation.

### Flags Wire Bytes

Add `wire_bytes: u8` field to `FlagsDef`. Computed from max bit position:
- Max bit 0–7 → 1 (u8)
- Max bit 8–15 → 2 (u16)
- Max bit 16–31 → 4 (u32)
- Max bit 32–63 → 8 (u64)
- No bits defined → 1 (u8, minimum)

Computed in `typeck.rs` alongside wire size computation.

### ResolvedAnnotations.deprecated Split

The current `ResolvedAnnotations` has `deprecated: Option<SmolStr>` which
stores only the `reason` string from `@deprecated(since: "v", reason: "r")`.
The `since` value is lost. To emit both `since` and `note` in
`#[deprecated(since = "...", note = "...")]`, change `deprecated` to a
structured type:

```rust
pub struct DeprecatedInfo {
    pub reason: SmolStr,
    pub since: Option<SmolStr>,
}
```

And change the field to `deprecated: Option<DeprecatedInfo>`. This requires
updating `lower.rs` to extract both `reason` and `since` from the annotation
arguments (similar to how `TombstoneDef` already extracts both).

## Testing Strategy

### Layer 1: `vexil-runtime` Unit Tests

In-crate tests for bit-level I/O primitives:

- BitWriter/BitReader round-trips: single bits, sub-byte, cross-byte-boundary,
  byte-boundary flushing
- LSB-first correctness: spec's worked example (u3 + u5 + u6 → exact bytes)
- LEB128: encode/decode, byte count limits, overlong rejection, edge cases (0, max)
- ZigZag: mapping correctness (0→0, -1→1, 1→2, -2→3), round-trip for each type width
- String/bytes: length prefix (max 4 LEB128 bytes), UTF-8 validation, global limit (2^26)
- Float: NaN canonicalization, negative zero preservation
- Empty message: single 0x00 byte from flush_to_byte_boundary
- Recursion depth: exceed 64 levels → RecursionLimitExceeded

### Layer 2: `vexil-codegen` Golden File Tests

Compile corpus `.vexil` → generate Rust → compare against checked-in snapshots:

```
crates/vexil-codegen/tests/golden/
  006_message.rs
  007_enum.rs
  008_flags.rs
  009_union.rs
  010_newtype.rs
  011_config.rs
  016_recursive.rs
```

Test runner does string comparison, reports diff on failure.

### Layer 3: Wire Format Round-Trip Tests

Hand-written Rust types + `Pack`/`Unpack` impls mirroring codegen output.
Tests prove wire format correctness without compiling generated code:

- Construct value → pack → unpack → assert equality
- Pack → assert exact byte sequence against spec-derived expected values
- Sub-byte packing: u3 + u5 + u6 produces `[0b_sssss_kkk, 0b_00_eeeeee]`
- Union wire format: discriminant + length prefix + payload
- Optional with byte-boundary flushing (byte-aligned T vs sub-byte T)
- Result: 1-bit discriminant, Ok and Err paths, void T/E
- Delta encoding: pack two sequential values, verify delta on wire
- Delta + varint composition: verify LEB128-encoded delta
- Delta + zigzag composition: verify ZigZag + LEB128-encoded delta
- Delta on f32/f64: verify float subtraction delta
- `@limit` enforcement on encode and decode
- `@non_exhaustive` enum: decode unknown → `Unknown(n)`
- `@non_exhaustive` union: decode unknown → skip via length prefix
- Overlong LEB128 → DecodeError::InvalidVarint
- Recursion depth exceeded → DecodeError::RecursionLimitExceeded
- Decode invalid input → correct error variant

## Scope Boundaries

### In Scope

- `vexil-runtime` crate (BitWriter, BitReader, Pack, Unpack, errors, LEB128, ZigZag)
- `vexil-codegen` crate (CompiledSchema → Rust source string)
- `vexilc codegen` subcommand
- All wire encoding rules from spec §3–§4
- All encoding annotations: `@varint`, `@zigzag`, `@delta` (including compositions and floats), `@limit`
- All codegen contract items from §11.2 except `SCHEMA_HASH`
- Tombstone comments + `REMOVED_ORDINALS` constants
- `@deprecated` (declaration-level and field-level), `@non_exhaustive`, `@doc`, `@since`
- Box insertion for recursive types through optional/result
- Recursion depth limit (64 levels) enforcement in BitReader
- IR additions: `EnumDef.wire_bits`, `FlagsDef.wire_bytes`, `DeprecatedInfo` split
- Three test layers

### Out of Scope (Deferred)

- `SCHEMA_HASH` — Milestone E (canonical form + BLAKE3). `DecodeError::SchemaMismatch` variant is defined but not emitted until Milestone E.
- Multi-file codegen (imported types) — Milestone F
- Other language backends — future
- `no_std` runtime support — future (design for it, don't implement)
- `@type`/`@domain`/`@revision` — VNP protocol-level; emitted as doc comments, not acted upon
