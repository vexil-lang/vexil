# Multi-File Project Example

Demonstrates Vexil's multi-file import system:

- **Cross-file imports** — `messages.vexil` imports types from `types.vexil`
- **Project compilation** — `vexilc build` resolves imports, deduplicates shared types, generates a module tree
- **Topological ordering** — dependencies are compiled in the correct order automatically

## Run

    cargo run

## Regenerate

    vexilc build schemas/project/messages.vexil --include schemas --output src

## Files

- `schemas/project/types.vexil` — shared type definitions
- `schemas/project/messages.vexil` — message types that import from types
- `src/project/` — Rust modules generated by `vexilc build`
- `src/main.rs` — uses types from both generated modules
